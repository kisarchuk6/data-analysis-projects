# data-analysis-projects
Есть список городов, хранящийся в таблице city:
Необходимо в каждом городе провести выставку книг каждого автора в течение 2020 года. Дату проведения выставки выбрать случайным образом. Создать запрос, который выведет город, автора и дату проведения выставки. Последний столбец назвать Дата. Информацию вывести, отсортировав сначала в алфавитном порядке по названиям городов, а потом по убыванию дат проведения выставок.

SELECT 
    name_city, 
    name_author, 
    /* Генерируем случайную дату: к 1 января прибавляем от 0 до 365 дней */
    DATE_ADD('2020-01-01', INTERVAL FLOOR(RAND() * 366) DAY) AS Дата
FROM 
    city, author
ORDER BY 
    name_city ASC, 
    Дата DESC;
 
 Структура мысли.
 Обычно мы соединяем таблицы по какому-то общему полю (например, id). Но здесь нам нужно обратное: чтобы каждая строка из таблицы city встретилась с каждой строкой из таблицы author.
Для этого используется либо оператор CROSS JOIN, либо просто перечисление таблиц через запятую в секции FROM.

Как получить случайную дату в 2020 году?
Начало: '2020-01-01'
Диапазон: 2020 год был високосным, в нем 366 дней
Случайность: Функция RAND() выдает число от 0 до 1. Если умножить его на 365 (разница в днях между 1 января и 31 декабря) и округлить, мы получим случайное число дней.
Сложение: Используем функцию DATE_ADD(), чтобы прибавить эти дни к началу года
